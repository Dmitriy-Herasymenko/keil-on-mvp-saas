# ТЗ Frontend: KeilOn MVP SaaS

# F1: Voice interface

## 1. Общая цель

Реализовать основной экран взаимодействия пользователя с ассистентом через голос в реальном времени.

Интерфейс отвечает за:

- захват аудио
- передачу аудио
- воспроизведение ответа
- отображение состояния

## 2. Основной сценарий

1. Пользователь открывает экран
2. Видит центральный элемент (сфера / кнопка)
3. Запускает взаимодействие
4. Начинает говорить
5. Интерфейс показывает “слушает”
6. После окончания речи:
    - “думает” (ожидание ответа)
7. При ответе:
    - воспроизводится голос
    - состояние “говорит”
8. Пользователь может перебить

## 3. UI состояния

Фронтенд отображает состояния, приходящие от системы:

- Listening — пользователь говорит
- Processing — ожидание ответа
- Speaking — ассистент говорит

⚠️ фронтенд не определяет состояние сам, а отображает его

## 4. Аудио

### Запись:

- запрос доступа к микрофону
- обработка отказа
- начало/остановка записи

### Передача:

- отправка аудио потока на бекенд
- поддержка непрерывного стриминга

### Воспроизведение:

- проигрывание аудио ответа
- поддержка потокового воспроизведения

## 5. Interrupt (перебивание)

При начале речи пользователя во время ответа:

- остановить воспроизведение аудио
- переключить UI в состояние Listening
- отправить событие на бекенд

⚠️ фронтенд не управляет логикой перебивания, только сигнализирует

## 6. UI элементы

- центральный элемент (сфера / кнопка)
- индикатор состояния
- кнопка микрофона (если используется)

## 7. Обработка ошибок

- отказ доступа к микрофону
- потеря соединения
- ошибки аудио

UI должен:

- отображать понятные сообщения
- оставаться работоспособным

## 8. Производительность

- минимальная задержка взаимодействия
- плавные анимации
- стабильная работа без зависаний

## 9. Адаптивность

- корректная работа на desktop и mobile
- корректная работа с микрофоном на мобильных устройствах

## 10. Принцип разграничения (критично)

Фронтенд:

- захватывает аудио
- отправляет поток данных
- воспроизводит ответ
- отображает состояния

Бекенд:

- обрабатывает аудио
- управляет диалогом
- принимает решения
- определяет состояния

## 11. Вне зоны фронтенда

- логика диалога
- обработка команд
- выбор моделей
- управление контекстом
- обработка interrupt
- генерация ответа

## 12. Результат

Пользователь может:

- начать голосовое взаимодействие
- говорить с ассистентом
- слышать ответ
- перебивать

👉 интерфейс работает стабильно и предсказуемо
